# 2026-03-21 RAG 迭代记录

## 本轮目标

这轮工作的重点是把 RAG 系统从“能跑通”推进到“更稳、更可解释、更适合持续优化”：

- 统一 Embedding 运行配置到 `text-embedding-v4`
- 建立评测集、对比模式和基线别名
- 引入低成本长文档评测
- 增加拒答策略和 citation 对齐优化
- 清理仓库里的临时和误生成文件

## 遇到的主要困难

### 1. Windows shell 与 WSL 输出编码不稳定

现象：

- 直接通过当前 Windows shell 调 WSL 命令时，中文输出出现乱码。
- `wsl` 还会额外打印 localhost 代理告警。

解决方式：

- 固定通过 `scripts/wsl_utf8.ps1` 包装 WSL 命令执行。
- 在 `AGENT_GUIDE.md` 里明确忽略 localhost 代理告警。

改进成效：

- 自测、日志查看和评测输出都能稳定读中文。
- 后续排障成本明显下降。

### 2. 实际运行配置和测试配置脱节

现象：

- 仓库代码已经补了 `text-embedding-v4` 兼容逻辑，但根目录 `.env` 实际仍然是 `text-embedding-v2`。
- 导致“测试像是在用 v4，正式运行仍可能回到 v2”。

解决方式：

- 把 `.env` 和 `.env.example` 默认值统一改成 `text-embedding-v4`。
- 同时把 README 里的默认配置全部改成 `v4 + 1536`。

改进成效：

- 以后默认运行就是 `v4`，不会再出现“代码支持 v4 但实际环境没切过去”的情况。

### 3. DashScope v4 默认维度与库表维度不一致

现象：

- `text-embedding-v4` 默认返回的维度不是当前库里使用的 `1536`。
- 直接切模型会报维度不匹配。

解决方式：

- 在 Embedding 请求里显式带 `dimensions=EMBEDDING_DIM`。
- 保持当前向量列和配置都用 `1536`。

改进成效：

- `v4` 能在不改库表维度的前提下稳定运行。
- 避免了“切模型就要立刻迁库”的风险。

### 4. DashScope v3/v4 的批量限制更严格

现象：

- 长文档入库时，Embedding 批次过大，接口会报批量上限错误。

解决方式：

- 按模型类型动态调整批大小。
- `v3/v4` 走更小的批量，老模型保留原有上限。

改进成效：

- 大文档入库不再因为单批过大直接失败。

### 5. 网关上传大 PDF 时触发 413

现象：

- 评测引入外部 PDF 后，网关层出现 `413 Request Entity Too Large`。

解决方式：

- 在 gateway 增加 `setClientMaxBodySize` 支持。
- 暴露 `GATEWAY_MAX_BODY_SIZE_MB` 环境变量。

改进成效：

- 长文档和大 PDF 上传更稳。
- 后续如果继续扩评测集，不需要先改代码再调体积限制。

### 6. `xlarge` 套件太烧免费额度

现象：

- 直接跑外部长文档全量评测时，免费 embedding 额度消耗过快。
- 继续把 `xlarge` 当日常回归不现实。

解决方式：

- 保留 `xlarge` 作为阶段性验收套件。
- 额外建立 `longlite`：从官方长文档摘录关键段落，保留长文档语义和结构信息，但大幅降低开销。

改进成效：

- 现在可以用 `small -> medium -> longlite` 做日常回归。
- 长文档方向仍然可测，但成本可控。

### 7. 拒答策略最初过于激进

现象：

- 初版 stub 判定把“短但有效的 section 正文”也当成了“只有标题/来源信息”。
- 导致 `small` 套件里正常问题被误拒答。

解决方式：

- 收紧 `source stub` 判定，只把“没有正文”或“正文基本就是 Source 行”的 chunk 当作空证据。
- 不再把“正文短”本身视为无效证据。

改进成效：

- 既能拒掉明显无关问题，又不会把真实短答案误杀。
- `small` 套件重新回到全绿。

### 8. Citation 容易贴到标题块而不是正文块

现象：

- 早期 citation 聚合更依赖 top item，常常先贴来源头块或标题块。
- 引用看起来“像对的”，但和答案正文不够贴合。

解决方式：

- citation 改成按“问题 + 最终答案”重排。
- 对只有 `Source:` 的 stub chunk 降权。
- 优先选择正文更充分、和答案关键词更贴近的 chunk 作为引用。

改进成效：

- 引用和答案的一致性更强。
- 例如 HTTP 定义问题现在会优先引用 `Abstract` 正文，而不是标题来源块。

## 本轮改进成效

### 运行配置

- 根目录 `.env` 已统一到 `text-embedding-v4`
- `.env.example` 已同步为 `v4 + 1536`
- README 与模板配置不再和真实运行脱节

### 评测体系

- `small`：`6/6`
- `medium`：`12/12`
- `large`：`18/18`
- `longlite`：`6/6`

说明：

- `xlarge` 仍保留，但不建议作为免费额度下的日常套件。

### 生成质量

- 新增 `refused / refusalReason / evidenceScore` 字段
- 不相关问题会明确拒答，而不是勉强作答
- citation 对齐更贴正文证据

## 当前建议的日常工作流

```bash
python scripts/run_eval.py --suite small
python scripts/run_eval.py --suite medium
python scripts/run_eval.py --suite longlite
```

如果需要阶段性验收再运行：

```bash
python scripts/run_eval.py --suite large
python scripts/run_eval.py --suite xlarge --request-timeout 120
```
